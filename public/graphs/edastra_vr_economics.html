What WebXR saves Edastra
No App Store fees on VR content
WebXR runs in the browser — outside Apple/Google store. Zero 30% store fee on VR subscriptions billed via web. At ₹5 Cr VR ARR, that saves ₹75L–1.5 Cr/year in fees alone.
Instant content updates · no review delay
WebXR content updates server-side instantly. Native app updates take 3–7 days for App Store review. For curriculum content tied to board exam dates, instant updates matter.
Device barrier drops to ₹500
Budget VR shells in India under ₹5,000 use the phone's screen and sensors to create stereoscopic 3D — compatible with WebXR. Irusu Play VR Plus at ₹3,000 works with any Android phone via Chrome.
Cross-platform: Quest + phone + desktop
WebXR works on VR headsets, AR-enabled smartphones, and desktops through Chrome, Edge, and Safari. Same Edastra VR content works on every device category from one URL.

AI 3D asset generation
  • Meshy AI — text/image → 3D model in minutes. A human cell, DNA helix, or volcano from a text prompt.
  • Luma AI — real object photos → 3D NeRF model. Scan a real classroom object, place it in VR.
  • Shap-E (OpenAI) — open-source 3D generation. Free to run, unlimited models.
  • Cost: ₹0–500 per model vs ₹2,000–5,000 for a 3D artist
Reuse 2D animation assets
  • Edastra's existing 2D animation pipeline produces sprites, textures, and character rigs that can be imported directly into Unity as billboards or sprites in 3D space.
  • A "hybrid VR" approach: 3D environment (AI-generated) + 2D characters (existing pipeline). Looks great, costs 2D.
  • First 50 VR modules can be built entirely from existing 2D content library — zero new asset cost.
A-Frame / Three.js over Unity
  • Unity developer = ₹1–1.5L/mo. A-Frame/Three.js web developer = ₹60–80K/mo. Same VR output, 40% lower salary.
  • WebXR with A-Frame uses HTML and JavaScript — your existing frontend developers can build VR modules, no game engine expertise required.
  • Open source, no royalty fees vs Unity's revenue share above $200k/yr.

Offline-first VR architecture
Module packaging
50–150MB per module
Each VR module packaged as a compressed asset bundle. 3D scenes, textures, audio, and WebXR code in a single downloadable ZIP. Average 5-min VR module = 80MB compressed.
→ gzip + Brotli compression · Draco mesh compression · KTX2 textures
Progressive download on WiFi
Background sync
App detects WiFi connection → downloads next 3 recommended modules in background. Student never waits. Service worker caches modules in IndexedDB for offline WebXR playback.
→ Service Worker · IndexedDB · Workbox.js
CDN used only for downloads
10x cheaper
CDN serves the initial download (one-time, 80MB). Not a live stream. Cloudflare R2 costs $0.015/GB storage vs $0.085/GB AWS S3 egress. One download per user per module. No repeat bandwidth.
→ Cloudflare R2 · ₹1.1/GB vs ₹7.5/GB AWS CDN
Additional infra savings
LOD (Level of Detail) rendering
3D scenes use high-res textures on Quest but automatically downsample to 50% resolution on phone VR. Same scene file, adaptive quality. No separate content builds.
Shared asset library
Common assets (human body, solar system, classroom environment) stored once on device, shared across all modules. A student downloads the "human body" 3D model once — it's used in biology, health, and science modules without re-downloading.
Server-side rendering fallback
For low-end phones, render the VR scene on a cloud GPU and stream as a video (like Google Stadia). Student sees full-quality VR without a powerful phone. Cost: ~₹2 per 30-min session on spot GPU instances.
Delta updates only
When a module is updated (new narration, corrected diagram), only the changed assets are re-downloaded — not the full 80MB. Reduces repeat bandwidth by 80–90% for content updates.
